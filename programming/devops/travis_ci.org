* Travis CI

file:../images/travis_ci/01.png

Travis CI 会对 repo 的所有分支的更行和 PR 运行测试，但是只在指定条件下运行部署，如 master 分支更新时。

* 配置

配置文件为 =.travis.yml= 。

** 生命周期

#+begin_src sh
before_install
install

before_script
script
    after_success
    after_failure
after_script

before_deploy
deploy
after_deploy
#+end_src

** =env=

设置环境变量。

#+begin_src yaml
env:
  global:
    - <name>=$(<expr>)
#+end_src

** =services=

#+begin_src yaml
services:
  - <string>
  - ...
#+end_src

** =sudo=

#+begin_src yaml
sudo: required
#+end_src

* 环境变量

密码等敏感信息可以设置为项目的环境变量，然后在配置文件中引用，如 =$DB_PASSWORD= 。
